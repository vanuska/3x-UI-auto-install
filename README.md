### Универсальный скрипт установки 3X-UI


# Универсальный установщик сервера с 3x-ui, Portainer и автоматической настройкой
Этот скрипт полностью автоматизирует развёртывание сервера для работы с 3x-ui (панель управления Xray), Portainer (управление Docker-контейнерами), Watchtower (автообновление контейнеров) и базовой безопасностью.
Скрипт можно запускть несколько раз без очистки сервера. Но не желательно запускать чаще чем ~5 раз подряд (приблизительно или закоментировав ссответствующую секцию скрипта выпуска SSL), так как выпуск сертификата может быть заблокирован на стороне Let's Encrypt.  

## Скрипт настраивает:

- SSH на порт 2233 с поддержкой двухфакторной аутентификации (Google Authenticator, Yandex ID, MS Authenticator и пр.) и запретом root-логина.
- Файрвол UFW с открытием только необходимых портов.
- Docker и Docker Compose.
- BBR для ускорения сети.
- Let's Encrypt сертификаты через acme.sh.
- 3x-ui в Docker с монтированием сертификатов.
- Portainer в одном из 6 режимов (полноценный сервер, Edge Agent Standard, Edge Agent Async, Legacy Agent, API, Socket).
- Watchtower для автоматического обновления образов.
- Ежедневные бэкапы и автоматическое обновление сертификатов.

## Особенности
- Интерактивный выбор режима установки Portainer (подходит для любых сценариев).
- Надёжный SSH — порт 2233 фиксируется через systemd drop-in, не сбрасывается при обновлениях.
- Полная изоляция — скрипт не перезаписывает системные файлы, использует только рекомендуемые методы.
- Автоматическое получение SSL-сертификатов для домена, указанного при установке.
- Всё в Docker — простота обслуживания и обновления.

## Требования
- Стабильный сервер с Ubuntu с минимальными ресурсами (20.04, 22.04 или 24.04 LTS).
- Публичный IP-адрес и домен, указывающий на этот IP - A-запись в DNS.
- Доступ к консоли (SSH или VNC) для первоначального запуска.
- Пользователь root или возможность выполнения команд с sudo.

## Использование
1. Скачайте скрипт на сервер и выполните от root:
```bash
wget -O install.sh https://github.com/vanuska/3x-UI-auto-install/tree/feature/standalone-or-agent-3x-ui/install.sh
```
или запустите напрямую:
```bash
sudo bash -c "$(curl -fsSL https://github.com/vanuska/3x-UI-auto-install/tree/feature/standalone-or-agent-3x-ui/install.sh)"
```
2. Сделайте исполняемым
```bash
chmod +x install.sh
```
3. Запустите
```bash
sudo ./install.sh
```
4. Следуйте подсказкам
Вам будет предложено:
- Ввести домен (например, my-server.ru). Домен должен иметь DNS A-запись на ваш IP. Например через Cloudflare, DNS only (для Proxied требуется дополнительная настройка, читайте отдельно). 
- Ввести имя пользователя (для создания системного пользователя, от которого будете работать).
- Выбрать тип установки Portainer (см. ниже).

После этого скрипт выполнит все шаги автоматически. 
Процесс может занять 5–10 минут.
По окночании, желательно, но не обязаткльно, перезагрузить сервер. 
Пароль будет сгенерирован автоматически и показан в конце установки.

## Выбор типа установки Portainer
Скрипт предлагает 6 вариантов:
#### <br>1	<--> Полноценный сервер	<-> Устанавливает Portainer CE с веб-интерфейсом на порту 9443 с SSL. Подходит для самостоятельного управления.

### <br>Для вариантов 2–4 будет запрошен адрес Portainer-сервера и токен/ключ.
#### <br>2	<--> Edge Agent Standard <-> Устанавливает агент, который подключается к удалённому Portainer через постоянный WebSocket-туннель. Требуется адрес сервера и ключ.
#### <br>3	<--> Edge Agent Async	<-> То же, но с периодической синхронизацией (без постоянного туннеля). Экономит трафик.
#### <br>4	<--> Legacy Agent	<-> Классический агент Portainer, подключается по HTTP/HTTPS к серверу на порту 9000. Открывает порт 9001 на агенте.
#### <br>5	<--> API <-> Не устанавливает агент. Выводит инструкцию по подключению через Docker API (рекомендуется для опытных).
#### <br>6 <--> Socket	<-> Не устанавливает агент. Выводит инструкцию по подключению через локальный Docker-сокет (подходит, если Portainer запущен на том же хосте).

### После установки
1. Настройка SSH
Порт: 2233
Пользователь: USERNAME, пароль сгенерирован и выведен в конце установки.
Важно! Войдите под этим пользователем и смените пароль:
```bash
su - USERNAME
passwd
```
Для включения двухфакторной аутентификации выполните:
```bash
google-authenticator
```
Следуйте инструкциям, затем перезапустите SSH-сокет:
```bash
sudo systemctl restart ssh.socket
```
2. Настройка 3x-ui
Первый вход: http://IP_СЕРВЕРА:3322/путь (путь вы получите после сброса через консоль 3x-ui, 10. Change port).
<br>Для сброса логина/пароля и пути выполните:
```bash
docker exec -it 3x-ui x-ui
```
Выберите пункты меню: 6. Reset Username & Password, 7. Reset Web Base Path, 10. Change port - укзать 3322.

После этого зайдите в панель и укажите сертификаты для самой панели и для подписок:
- Certificate: /root/cert/fullchain.pem
- Private Key: /root/cert/privkey.pem

3. Portainer
Если выбран режим сервер → доступен по https://ВАШ_ДОМЕН:9443/.
<br>Если выбран агент → подключите его в вашем основном Portainer, используя выданный ключ.
<br>При использовании API или Socket следуйте инструкциям, выведенным в конце установки.

4. Watchtower
Автоматически обновляет контейнеры раз в сутки (интервал 86400 секунд).
<br>Логи можно посмотреть:
```bash
docker logs watchtower
```
6. Бэкапы
Ежедневно в 4:15 создаётся архив /opt/3x-ui/backup/3x-ui-дата.tar.gz.
Хранятся 14 дней, старые удаляются автоматически.
<br>Запуск вручную:
```bash
/opt/3x-ui/backup.sh
```
6. Обновление сертификатов
Каждую ночь в 3:58 выполняется скрипт /usr/local/bin/ssl-renew.sh.
Если сертификат обновлён, перезапускаются Portainer и 3x-ui.
<br>Запуск вручную:
```bash
sudo /usr/local/bin/ssl-renew.sh
```
## Примечание: стек 3x-ui создан с ограниченным через Portainer.io контролем. 
<br>Для полного управления через Portainer удалите стек через Container и добавьте его заново через интерфейс Portainer - Stacks, скопировав в него содержимое из /opt/stacks/3x-ui/docker-compose.yml. 
<br>То же самое и для Watchtower.

### Устранение неполадок
SSH не подключается на порт 2233
Проверьте статус сокета:
```bash
systemctl status ssh.socket
ss -tlnp | grep 2233
```
Убедитесь, что UFW разрешает порт:
```bash
ufw status | grep 2233
```
Если порт снова стал 22 – вероятно, обновление пакета перезаписало системный файл. В данной версии скрипт теперь использует drop-in, поэтому этого не произойдёт.

3x-ui не запускается
Проверьте логи:
```bash
docker logs 3x-ui
```
Убедитесь, что сертификаты смонтированы корректно:
```bash
ls -la /opt/portainer-cert/
docker exec 3x-ui ls -la /root/cert/
```
Ошибки REALITY при подключении клиента:
- В панели 3x-ui для inbound на порту 443 выберите security: tls (не reality).
  В клиенте используйте обычный TLS без полей reality.
- Не открывается Portainer.
  Проверьте контейнер:
  ```bash
  docker ps -a | grep portainer
  ```
  Логи:
  ```bash
  docker logs portainer
  ```
  Если сертификаты недействительны, обновите их вручную:
  ```bash
  /usr/local/bin/ssl-renew.sh
  ```
