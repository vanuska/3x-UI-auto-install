### Универсальный скрипт установки 3X-UI

Скрипт автоматически настраивает Ubuntu сервер с Docker, панелью 3x-ui, Portainer, Watchtower, SSL сертификатами и двухфакторной аутентификацией SSH.

Для получения SSL сертификата, предварительно, вам необходимо создать A-запись на DNS - сопоставление VDS IP с вашим доменом, к примеру 3-го уровня.

Сертификт будет обновляться на уровне ОС, не на уровне Докера\Stacks.

В скрипте можно изменть порты и Timezone на свои значения. 

## Запуск

Скачайте и выполните скрипт от root:
```bash
wget https://github.com/vanuska/3x-UI-auto-install/install.sh
chmod +x install.sh
sudo ./install.sh
```

Или запустите напрямую:
```bash
sudo bash -c "$(curl -fsSL https://github.com/vanuska/3x-UI-auto-install/install.sh)"
```

Во время установки скрипт запросит доменное имя и имя пользователя. Пароль будет сгенерирован автоматически и показан в конце.

# Что устанавливается

## Базовые пакеты: 
curl, wget, nano, mc, htop, git, sudo, ufw, socat, dnsutils, ca-certificates, openssl, libpam-google-authenticator.
## Docker и компоненты: 
docker-ce, docker-ce-cli, containerd.io, docker-buildx-plugin, docker-compose-plugin.
## Сервисы в Docker: 
Portainer (порт 9443) для управления контейнерами, Watchtower для автоматического обновления образов, 3x-ui (порт 3322) для управления Xray.

## Настройка безопасности

<br>SSH переносится на порт 2233.
<br>Вход только по паролю с обязательной двухфакторной аутентификацией через Google Authenticator.
<br>Запрещён вход под root.
<br>Настроен файрвол UFW, открыты порты 2233, 3322, 3333, 443, 9443.
<br>SSL сертификаты выпускаются через acme.sh и автоматически обновляются каждый день.
<br>Ежедневно создаётся резервная копия конфигураций в /opt/3x-ui/backup, старые копии удаляются через 14 дней.

## После установки

<br>Проверьте вход по SSH в отдельном окне терминала, используя порт 2233.
<br>Смените пароль пользователя командой passwd.
<br>Настройте двухфакторную аутентификацию командой google-authenticator.
<br>Настройте 3x-ui: ВНИМАНИЕ НОМЕРА ПУНКТОВ в заисимости от релиза могут отличаться по цифрам! 
<br>Выполните: 
```bash 
docker exec -it 3x-ui sh
```
затем 
```bash
x-ui
```
выберите пункты 6 (сброс логина/пароля) - запомнить, 7 (сброс пути веб-интерфейса) - скопировать, 10 (смена порта) на 2233.
<br>Первый раз заходим в web-интерфейс 3x-ui по IP https://IP:2233 или http://IP:2233 и указываем пути к сертификатам 
```bash
/root/cert/fullchain.pem
```
и 
```bash
/root/cert/privkey.pem
```
<br>Далее в разделе "Подписки" порт измените на 3333, а путь URI на свой, любой выдуманный.
<br>После сохранения настроек перезапустите панель и заходите по адресу https://ВАШ-ДОМЕН:2233/СГЕНЕРИРОВАННЫЙ ПУТЬ и п7.

## Настройка Portainer.io
https://ВАШ-ДОМЕН:9443/
Если portainer запросит TOKEN , то выполням
```bash
docker ps
```
копируем номер контейнера portainer 
```bash
docker logs НОМЕРPORTAINERа который скопировали
```

## Примечание: стек 3x-ui создан с ограниченным через Portainer.io контролем. 
<br>Для полного управления через Portainer удалите стек через Container и добавьте его заново через интерфейс Portainer - Stacks, скопировав в него содержимое из /opt/stacks/3x-ui/docker-compose.yml. 
<br>То же самое и для Watchtower.
